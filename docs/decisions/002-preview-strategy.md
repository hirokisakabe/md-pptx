# プレビュー方式：スライドマスター背景画像の合成

## ステータス

置き換え済み（004-pptx-glimpse に移行）

## コンテキスト

VS Code 拡張でのリアルタイムプレビューを実現するにあたり、テンプレート PPTX の見た目をどのように再現するかが課題だった。プレビューのたびに PPTX 生成を行うとパフォーマンスが低下するため、軽量な代替手段が必要だった。

## 決定

テンプレート PPTX からスライドマスターの背景画像を抽出し、Webview の背景に敷く方式を採用した。PPTX 生成は明示的なコマンド実行時のみ行い、プレビューでは行わない。

## 理由

- **速度優先**: PPTX 生成を伴わないため、リアルタイムプレビューに適したパフォーマンスを確保できる
- **実装の容易さ**: JSZip でテンプレート PPTX を展開し、スライドマスターの背景画像を Data URL に変換するだけで実現できる
- **実用上十分な体験**: テンプレートのプレースホルダ位置を初回に読み取り、その座標に合わせてコンテンツを描画する。完全一致はしないが、実用上十分な見た目を提供する

## リスク

- 図形やグラデーションで構成された背景はラスタライズが必要であり、単純な画像背景にのみ対応している
- プレースホルダの座標に基づく描画は近似であり、最終出力の PPTX とレイアウトが完全には一致しない
