# Marp 互換性の範囲

## ステータス

採用済み

## コンテキスト

Markdown からスライドを生成するツールとして Marp が広く使われている。md-pptx の Markdown 記法を設計するにあたり、Marp との互換性をどの程度持たせるかを決める必要があった。

## 決定

Marp の記法をベースとしつつ、md-pptx の目的に合わない機能は非対応とした。

互換としたもの：

- スライド区切り（`---`）
- Front Matter による設定（`headingDivider` 等）
- HTML コメント形式のディレクティブ（`<!-- directive: value -->`）

非対応としたもの：

- CSS によるスタイリング（Marp はテーマ CSS でデザインを制御するが、md-pptx はテンプレート PPTX がデザインを担う）
- HTML/PDF 出力（md-pptx は編集可能な PPTX 出力に特化）

## 理由

- **学習コストの低減**: Marp ユーザーが既存の知識を活かして移行できる
- **明確な差別化**: Marp が「wontfix」としたネイティブ PowerPoint 出力を、python-pptx-wasm で実現するという立ち位置を明確にする
- **テンプレート PPTX 中心の設計**: デザインの制御はテンプレート PPTX のスライドマスターに委ねるため、CSS テーマの仕組みは不要

## リスク

- Marp の記法を期待するユーザーが、非対応の機能に遭遇して混乱する可能性がある
- Marp 側の記法変更に追従する必要が生じる可能性がある
