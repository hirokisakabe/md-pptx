# pptx-glimpse の採用

## ステータス

採用済み（002-preview-strategy を置き換え）

## コンテキスト

VS Code 拡張のリアルタイムプレビューでは、当初スライドマスターの背景画像を抽出して Webview の背景に敷く方式を採用していた（002-preview-strategy）。この方式には以下の課題があった：

- 図形やグラデーションで構成された背景を再現できない
- プレースホルダ座標に基づく近似描画のため、最終出力の PPTX とレイアウトが一致しない
- テキストの書式（フォントサイズ、色、配置等）を忠実に再現できない

プレビューの忠実度を向上させるため、PPTX をより正確にレンダリングできるライブラリの導入を検討した。

## 決定

pptx-glimpse を採用し、PPTX → SVG 変換によるプレビュー方式に移行した。生成した PPTX を pptx-glimpse で SVG に変換し、Webview に表示する。

## 理由

- **プレビュー忠実度の向上**: PPTX を直接 SVG に変換するため、背景画像の抽出と座標の近似に頼る旧方式と比べ、最終出力に近いプレビューを提供できる
- **実装の簡素化**: 背景画像抽出・プレースホルダ座標読み取り・コンテンツ描画といった独自実装が不要になり、`convertPptxToSvg` の呼び出しに集約できる
- **メンテナンスコストの削減**: プレビューのレンダリングロジックをライブラリに委譲することで、テンプレート形式の変化や新しい要素への対応をライブラリ側のアップデートで吸収できる

## リスク

- pptx-glimpse は発展途上のライブラリであり、対応していない PPTX 要素がある。必要に応じて PR で貢献またはフォークで対応する方針
- PPTX 生成を伴うプレビューとなるため、旧方式（背景画像のみ）と比べてプレビュー更新のパフォーマンスが低下する可能性がある
